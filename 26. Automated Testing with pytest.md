### 3. Writing Your First Test

Create a new file called `test_calculator.py`:

```python
# test_calculator.py

def add(a, b):
    return a + b

def divide(a, b):
    if b == 0:
        raise ValueError("Cannot divide by zero")
    return a / b


# Tests
def test_add():
    assert add(2, 3) == 5
    assert add(-1, 1) == 0
    assert add(0, 0) == 0

def test_divide():
    assert divide(10, 2) == 5
    assert divide(9, 3) == 3

def test_divide_by_zero():
    try:
        divide(10, 0)
    except ValueError:
        assert True
    else:
        assert False, "Should have raised ValueError"
```

---

### 4. Running Tests

Run this command in your terminal:

```bash

```


### 5. More pytest Features

```python
import pytest

# Testing with multiple inputs
@pytest.mark.parametrize("a, b, expected", [
    (2, 3, 5),
    (0, 5, 0),
    (-4, 2, -2),
])
def test_add_parametrized(a, b, expected):
    assert add(a, b) == expected


# Testing exceptions
def test_divide_by_zero_exception():
    with pytest.raises(ValueError):
        divide(10, 0)
```

### 6. Testing a Real Function (from previous lessons)

```python
# test_expense.py
from datetime import datetime

def calculate_total(expenses):
    return sum(expense["amount"] for expense in expenses)

def test_calculate_total():
    sample_expenses = [
        {"amount": 100},
        {"amount": 250},
        {"amount": 75}
    ]
    assert calculate_total(sample_expenses) == 425
```

---

### 📝 Notes

- Test file names must start with `test_` or end with `_test.py`
- Test function names must start with `test_`
- Use `assert` to check if conditions are true
- `pytest.raises()` is used to test exceptions
- `@pytest.mark.parametrize` helps test multiple cases efficiently
