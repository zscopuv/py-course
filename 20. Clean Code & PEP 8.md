### 1. What is Clean Code?

Clean code is code that is:
- Easy to read and understand
- Easy to maintain and modify
- Well-organized and consistent

---

### 2. PEP 8 – The Official Python Style Guide

**PEP 8** is the official style guide for Python. Following it makes your code look professional.

#### Key PEP 8 Rules:

```python
# ✅ Good (Follows PEP 8)
def calculate_total_price(items):
    total = 0
    for item in items:
        total += item.price
    return total


# ❌ Bad
def calculatetotalprice(Items):
    total=0
    for i in Items:
        total+=i.price
    return total
```

---

### 3. Important PEP 8 Rules

| Rule                        | Good Example                          | Bad Example                     |
|----------------------------|---------------------------------------|---------------------------------|
| Indentation                | 4 spaces                              | Tabs or 2 spaces                |
| Line Length                | Max 79 characters                     | Very long lines                 |
| Variable Naming            | `user_name`, `total_price`            | `x`, `data1`, `UserName`        |
| Function Naming            | `get_user_data()`                     | `GetUserData()`                 |
| Class Naming               | `ExpenseTracker`                      | `expensetracker`                |
| Constants                  | `MAX_RETRIES = 5`                     | `maxretries = 5`                |
| Spacing                    | `x = 5 + 3`                           | `x=5+3`                         |

### 4. More Best Practices

```python
# 1. Meaningful Names
age = 25                    # Bad
user_age = 25               # Good

# 2. Docstrings
def calculate_bmi(weight_kg: float, height_m: float) -> float:
    """Calculate BMI given weight in kg and height in meters."""
    return round(weight_kg / (height_m ** 2), 2)


# 3. Type Hints (Modern Python)
def greet(name: str) -> str:
    return f"Hello, {name}!"


# 4. Avoid Magic Numbers
MAX_LOGIN_ATTEMPTS = 3      # Good

for attempt in range(MAX_LOGIN_ATTEMPTS):
    ...
```

### 5. Practical Example – Before vs After

**Before (Messy Code):**
```python
def calc(a,b,c):
 x=a+b*c
 if x>100:print("big");return x
 else:return 0
```

**After (Clean Code):**
```python
def calculate_final_score(base_score: int, bonus: int, multiplier: int) -> int:
    """Calculate final score with bonus and multiplier."""
    final_score = base_score + (bonus * multiplier)
    
    if final_score > 100:
        print("Excellent performance!")
        return final_score
    return 0
```

---

### 📝 Notes

- Always use **4 spaces** for indentation.
- Keep lines under **79 characters**.
- Use meaningful names (be descriptive).
- Write docstrings for functions and classes.
- Keep functions small and focused (one job per function).
- Use blank lines to separate logical sections.
