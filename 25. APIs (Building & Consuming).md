### 1. What is an API?

An API (Application Programming Interface) allows different applications to communicate with each other. You send a request and get data back (usually in JSON format).

---

### 2. Consuming Public APIs using `requests`

```python
import requests

# Example: Get random user data
response = requests.get("https://randomuser.me/api/")

if response.status_code == 200:
    data = response.json()
    user = data['results'][0]
    
    print(f"Name: {user['name']['first']} {user['name']['last']}")
    print(f"Email: {user['email']}")
    print(f"Country: {user['location']['country']}")
else:
    print("Failed to fetch data")
```

### 3. Building Your Own API with Flask

Update your `app.py`:

```python
from flask import Flask, jsonify, request

app = Flask(__name__)

# Sample data
books = [
    {"id": 1, "title": "Python for Beginners", "author": "John Doe"},
    {"id": 2, "title": "Flask Web Development", "author": "Jane Smith"}
]

# GET - Return all books
@app.route("/api/books", methods=["GET"])
def get_books():
    return jsonify({"success": True, "books": books})

# GET - Return single book by ID
@app.route("/api/books/<int:book_id>", methods=["GET"])
def get_book(book_id):
    book = next((book for book in books if book["id"] == book_id), None)
    if book:
        return jsonify({"success": True, "book": book})
    return jsonify({"success": False, "error": "Book not found"}), 404

# POST - Add new book
@app.route("/api/books", methods=["POST"])
def add_book():
    new_book = request.get_json()
    new_book["id"] = len(books) + 1
    books.append(new_book)
    return jsonify({"success": True, "message": "Book added", "book": new_book}), 201

if __name__ == "__main__":
    app.run(debug=True)
```

---

### 4. Testing Your API

After running the app, test these URLs:

- `http://127.0.0.1:5000/api/books` → Get all books
- `http://127.0.0.1:5000/api/books/1` → Get one book
- Use **Postman** or **Thunder Client** (VS Code) to test POST requests

---

### 📝 Notes

- `jsonify()` converts Python dictionaries into proper JSON responses.
- Use `request.get_json()` to receive data from POST requests.
- Always return meaningful success/error messages.
- Status codes matter: `200` = success, `201` = created, `404` = not found.
